Lab Exam Practice – HTML Foundations

Part A: Find Mistakes (Identify + Fix)

1. Navigation menu structure

Mistake: A link is placed directly inside <ul> instead of inside <li>.

<nav aria-label="Primary navigation">
  <ul>
    <li><a href="index.html">Home</a></li>
    <li><a href="cv.html">CV</a></li>
  </ul>
</nav>

2. Form label association

Mistake: The for attribute does not match the control’s id.

<label for="message">Message</label>
<textarea id="message" name="message"></textarea>

3. Heading hierarchy

Mistake: Heading levels are skipped.

<h1>Interaction Analysis</h1>
<h2>Norman vs GOMS</h2>

4. Accessible table structure

Mistake: Table lacks caption and header cells.

<table>
  <caption>Models and what they predict</caption>
  <thead>
    <tr>
      <th scope="col">Model</th>
      <th scope="col">Predicts</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <th scope="row">KLM</th>
      <td>Execution time</td>
    </tr>
  </tbody>
</table>

5. aria-current usage

Mistake: More than one link is marked as the current page.

<nav aria-label="Primary navigation">
  <ul>
    <li><a href="index.html">Home</a></li>
    <li><a href="cv.html" aria-current="page">CV</a></li>
    <li><a href="contact.html">Contact</a></li>
  </ul>
</nav>

Part B: True / False

Statement Answer
Every link inside <ul> must be inside an <li> True
A page may have multiple <h1> headings to make text larger False
aria-current="page" should be used on the active navigation item True
A form is accessible without any labels False
KLM predicts execution time for expert users True

Part C: Short Answers

Semantic vs Generic Elements

Semantic elements describe their meaning and role clearly to browsers and assistive technologies, while generic elements have no inherent meaning and are used only for layout or grouping.

Why KLM does not model planning

KLM focuses on low-level physical actions and assumes that planning, decision-making, and interpretation have already been completed by the user.
