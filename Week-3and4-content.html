Week 3: Interface Design & Usability Principles

Key Principles

Usability Principles
Principle Definition Example in Interfaces
Consistency Design elements behave the same way across system Menus, buttons, and labels have uniform style
Feedback Provide system responses to user actions Loading bars, confirmation messages
Error Prevention Reduce chances of mistakes Disabled buttons until required fields filled
Recognition over Recall Minimize user's memory load by showing options Dropdown menus with pre-filled suggestions

Interface Chunking Example

Examples of Interface Chunking
Example Observation Key Point
YouTube Side panel groups account/search, navigation, personal items Reduces cognitive load, users focus easily
Canva Dashboard separates features into labeled sections Supports focus on one task at a time
DataCamp Lessons organized into small sections Supports STM, prevents cognitive overload
Etsy Homepage content grouped into labeled categories Users process a limited number of items at once

Week 4: Cognitive Load, Affordances, & Interaction Styles

Cognitive Load

Cognitive Load Types
Type Definition Design Implication
Intrinsic Complexity inherent to the task Break complex tasks into smaller steps
Extraneous Unnecessary cognitive effort caused by interface Remove clutter, reduce irrelevant info
Germane Resources used for learning and understanding Provide cues, examples, and summaries

Affordances & Signifiers

Affordances in Interface Design
Concept Definition Example
Affordance Properties that suggest how something can be used Button appears clickable
Signifier Signal that communicates how to use an element Arrow icon indicating scroll direction

Interaction Styles

Types of Interaction Styles
Style Definition Example
Command-Line Text-based input/output interface Terminal or console
Menu Selection User selects options from menus Website dropdown navigation
Form-Fill Users input data into fields Login or signup forms
Direct Manipulation Users interact directly with objects Dragging files in a folder
